# Naming, Versioning, and File Conventions

<!-- 这是一个 sample，文件实质完成后删掉这行注释 -->

Directories and prose documents use descriptive kebab-case names. Python files use descriptive snake_case names. Stable tool-recognized files retain conventional names. Source models, derived-author models, runbooks, schemas, and runtime adapters use semantic versions. Work-item identifiers use `YYYY-NNN-descriptive-slug`.
